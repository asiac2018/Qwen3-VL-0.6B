## 🏗️ 架构设计

```
┌─────────────────────────────────────────────────────┐
│                    SiQ-VL 架构                       │
│                                                     │
│  Image (224×224)                                    │
│       ↓                                             │
│  SigLIP2-so400m-patch14-224 (430M, 冻结)            │
│       ↓ 256 vision tokens (1152-dim)                │
│  2-Layer MLP + Pixel Shuffle (factor=2)             │
│       ↓ 64 vision tokens (1024-dim)                 │
│  Qwen3-0.6B (600M, Stage2 LoRA r=128)              │
│       ↓                                             │
│  Text Response                                      │
└─────────────────────────────────────────────────────┘
```

**关键设计决策**：
- **Pixel Shuffle**: 将 256 个视觉 token 压缩至 64 个，大幅减少计算量
- **2-Layer MLP Projector**: 比单层 Linear 更强的对齐能力 (5.8M 参数)
- **两阶段训练**: Stage 1 只训 Projector (冻结视觉+文本), Stage 2 用 LoRA 微调文本模型

### 反幻觉突破点 🔑

v4 通过每张图片 3 个否定样本 (询问图片中**不存在**的物体) 实现了反幻觉从 0/3 → 3/3 的突破：
- "Is there an airplane?" → "No, there is no airplane in this image." ✅

### v3 文本退步的教训 🔑

- v3 使用简洁答案格式 (如 "4.") 导致文本推理能力严重退步 (100% → 62.5%)
- v4 恢复详细文本答案 (如 "2+2 equals 4. This is a basic addition problem.") 后恢复至 87.5%
- **核心经验**: 视觉训练数据的格式会影响文本能力，必须保持文本训练的答案详细度

## 🔑 你需要了解的核心关键点

### 1. Qwen3 与 Qwen2 不兼容 — 代码必须修改

- `Qwen2ForCausalLM` → **`Qwen3ForCausalLM`** (Qwen3 有 q_norm/k_norm, 无 attention bias)
- 共 56 个权重 key 受影响，直接用 Qwen2 代码会导致权重加载失败

### 2. eos_token_id 配置错误 — 影响生成终止

- siq-vl 默认 `eos_token_id=151643` (空 token)
- 正确应为 `eos_token_id=[151643, 151645]` (151645 = `<|im_end|>`)
- 否则模型生成无法正常终止

### 3. LoRA 权重合并的坑 — 无法直接 merge_and_unload

- 自定义 `SiQ_VLForCausalLM` 不是标准 PeftModel，`merge_and_unload()` 会报错
- **解决方案**: 手动构建 PeftModel → 手动映射 key → 加载权重 → 再 merge
- Key 映射规则: `text_model.base_model.model.*` → `base_model.model.text_model.*`
- 992/1154 个 key 可映射 (162 个视觉模型 key 已存在于 base model)

### 4. 训练数据质量 > 数量

- 仅用 ~1000 样本就实现了基本视觉能力
- 否定样本是反幻觉的关键 (v4 每图 3 个否定)
- SOTA VLM 用 600K+ 样本，当前数据量是最大瓶颈

### 5. 生成参数推荐

```python
temperature=0.1          # 事实性问题用低温度
repetition_penalty=1.3   # >1.5 会导致乱码, <1.2 会重复
top_k=20
max_new_tokens=80        # 控制长度以减少重复
eos_token_id=[151643, 151645]
```

## 💡 核心结论

> **项目已成功将纯文本 Qwen3-0.6B 升级为视觉语言模型**, 在有限训练数据 (~1000 样本) 下实现了物体识别、颜色识别 (70%)、反幻觉 (100%)、计数等核心视觉能力，同时保持了 87.5% 的文本推理能力。
>
> **最大瓶颈是训练数据量** — SOTA VLM 使用 600K-1.2M 样本，当前仅 1K。恢复网络访问后，数据量的提升将带来最显著的性能飞跃。
